About this UI

What it is

A lightweight, project-specific web UI that reads the DAG files via AST (Abstract Syntax Tree) analysis and renders each one's task graph with what each task does. Built because the Airflow webserver shows runtime state, but for a learning project the structure of each DAG is the more useful view.

How extraction works

  1. For each *.py file under dags/, the extractor parses the source as Python AST (no execution).
  2. It locates the top-level with DAG(...) block and pulls out dag_id, schedule, description, and tags.
  3. It walks the DAG body, finding operator instantiations (PythonOperator(...), BashOperator(...), etc.) and TaskFlow @task calls. Each operator becomes a node in the graph.
  4. For each PythonOperator, it resolves the python_callable name to the function's docstring, giving every node a one-line activity description.
  5. It collects TaskGroup contexts and renders them as subgraphs.
  6. It walks >> chains (and lists inside them), flattening left-associative a >> b >> c into edges.
  7. The collected metadata becomes a Mermaid graph TD string, color-coded by operator type.

Why this approach

  • No side effects. Importing a DAG would trigger Airflow's DAG-bag registration. AST is pure.
  • No Airflow dependency at UI time. A learner can install only Flask and explore the project before ever running airflow db migrate.
  • Best-effort. Patterns the extractor can't capture statically (TaskFlow .expand(), branch-returned task ids, dynamic fan-out) generate a warning rather than failing. The DAG itself still works at runtime.

Run it

pip install flask
python main.py ui
# open http://localhost:5000

Mermaid.js is loaded from a CDN. The UI itself is just Jinja templates — no build step.
